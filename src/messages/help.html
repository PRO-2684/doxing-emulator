== 开盒模拟器使用说明 ==
您可以通过下列任一方式对他人进行开盒： ● 提供用户 ID：若 /dox 指令后为一串数字，则开盒此 ID 的用户。 ● 回复或引用消息：若 bot 在群聊中，使用 /dox 指令回复某人的消息即可对 ta 进行开盒。亦可在私聊中用 /dox 指令引用其它地方的消息。 ● 转发消息：将某人的消息转发至此机器人，即可对 ta 进行开盒。 ● Inline 使用：在任意聊天界面提及此 bot，并提供用户 ID，则开盒此 ID 的用户。 补充说明： ● 如果未指定目标，则是开自己的盒。 ● bot 的成功查询设有三分钟的缓存，同时 Telegram 服务器会有一分钟的缓存，因此您可能需要等待几分钟后才能查询到更新的内容。
== 错误代码 ==
● ERR_DOXEE_IDENTIFICATION_FAILED: 无法确定被开盒者的身份。如果是回复的消息，可能对方是匿名状态；如果提供了参数，可能用户不存在，或者 bot “不认识”此用户。 ● ERR_DOXER_IDENTIFICATION_FAILED: 无法确定指令发送者的身份。您可以向 bot 发送 /start 指令后等待 60s 再次尝试。 ● ERR_INCOMPREHENSIBLE: 无法理解的消息。消息既不是转发或引用的，也不是指令。 ● ERR_INVALID_ORIGIN: 转发或引用来源不是用户，或者被隐藏。 ● ERR_NOT_USER_ID: 提供的参数不是用户 ID。正确的用户 ID 应当由一串阿拉伯数字构成，并且是合法的 u64。
== 让 bot 认识某用户 ==
Telegram 出于隐私考虑，只允许 bot 获取与它互动过的用户的信息。为了让 bot 认识某用户，您可以： ● 转发/引用消息: 如果您可以带来源转发此用户的消息，则只需将 ta 的消息转发或引用到此 bot。 ● 让 ta 联系 bot: 如果您无法带来源转发，则需要 ta 向此 bot 发送一条消息，通常来说是 /start 指令。 ● 让 ta 点击链接: 您可以尝试让 ta 点击 tg://resolve?domain={username}&start=help 或 t.me/{username}?start=help 这两个链接之一。
